3. COMODI Description back to ToC

COMODI basically consists of 6 types of classes and 5 object properties linking them. As it describes changes, there is the central comodi:Change class representing a change in a delta between two versions of a computational model. The comodi:Change class is connects to 5 other types of classes using 5 different object properties:

  • comodi:actedOn comodi:Entity — linking the change to a comodi:Entity, which is for example an XML attribute. (see e.g. comodi:Attribute)
  • comodi:intended comodi:Intention — explaining the intention of a change, such as an extension of the model. (see e.g. comodi:Extension)
  • comodi:hadReason comodi:Reason — providing a reason for the change, such as a mismatch with the corresponding publication. (see e.g. comodi:MismatchWithPublication)
  • comodi:changeType comodi:Type — giving information about the type of change, such as an update. (see e.g. comodi:Update)
  • comodi:affected comodi:Target — indicating which part of the model has changed, such as a species definition. (see e.g. comodi:SpeciesDefinition)
