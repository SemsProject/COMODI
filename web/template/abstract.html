Abstract

The COMODI ontology is needed because COmputational MOdels DIffer. COMODI provides a set of classes and properties that empowers users and software to represent and describe changes in computational models on the semantic level. COMODI also enables software to implement user-specific filter options for the display of model changes. COMODI is the next step towards predicting how a change in a model influences the simulation study.

The OWL encoding of the COMODI Ontology is available from our web server.
The namespace for all COMODI terms is http://purl.org/net/comodi#.
