COMODI basically consists of 6 types of classes and 5 object properties linking them. As it describes changes, there is the central comodi:Change class representing a change in a delta between two versions of a computational model. The comodi:Change class is connects to 5 other types of classes using 5 different object properties:

  • comodi:appliesTo comodi:Entity — linking the change to a comodi:Entity, which is for example an XML attribute. (see e.g. comodi:Attribute)
  • comodi:hasIntenion comodi:Intention — explaining the intention of a change, such as an extension of the model. (see e.g. comodi:Extension)
  • comodi:hasReason comodi:Reason — providing a reason for the change, such as a mismatch with the corresponding publication. (see e.g. comodi:MismatchWithPublication)
  • comodi:hasChangeType comodi:Type — giving information about the type of change, such as an update. (see e.g. comodi:Update)
  • comodi:affects comodi:Target — indicating which part of the model has changed, such as a species definition. (see e.g. comodi:SpeciesDefinition)
In addition, comodi:Changes can be triggered by other comodi:Changes. For example, the deletion of a node triggers the deletion of all its attribute.
